FLASHCARD-MOODIN SÄÄNNÖT:

PAKOLLINEN MUOTO:
{
  "type": "flashcard",
  "question": "selkeä, suora kysymys",
  "answer": "tiivis ydinvastaus",
  "explanation": "2-3 hieman aiempaa informatiivisempaa, oppimista tukevaa lausetta"
}

TÄRKEÄÄ:
- Käytä VAIN tyyppiä: flashcard
- Yksi käsite/sääntö per kortti
- Ei täydennysviivoja ("___")
- Ei multiple_choice-, true_false-, matching- tai sequential-muotoa
- Pysy annetussa materiaalissa

KYSYMYS:
- Muotoile aktiivista muistamista tukeva suora kysymys.
- Hyvä runko: "Mikä on...?" "Mitä tarkoittaa...?" "Miten muodostetaan...?"

VASTAUS:
- answer: lyhyt ydinvastaus
- explanation: 2-3 lyhyttä mutta hieman aiempaa informatiivisempaa lausetta, jotka auttavat muistamaan oikean vastauksen
- Selitys kertoo ensin mitä pitää muistaa, sitten merkityksen, säännön, muistivinkin tai lyhyen esimerkin.
- Pidä selitys napakkana: tarpeeksi pitkä vahvistamaan muistamista, mutta tarpeeksi lyhyt nopeaan kertaukseen.
- ÄLÄ kirjoita quiz-tyylistä palautetta kuten "oikein vastaus", "miksi tämä on oikein" tai "yleinen virhe".
- ÄLÄ rakenna selitystä virheiden tai kompastuskohtien varaan.

SÄÄNTÖPOHJAINEN FLASHCARD-MUOTO (matematiikka, kielioppi, kaavat):
- Kysy sääntöä/periaatetta, ei yksittäistä koetehtävää.
- answer antaa kaavan/säännön.
- explanation vahvistaa säännön muistamista muistivihjeellä tai hieman aiempaa täydemmällä toimivalla esimerkillä.

VARMISTUS ENNEN VASTAUSTA:
- Jokainen kortti on validi flashcard JSON-rivi.
- Kaikissa korteissa on topic ja skill (tulee core-säännöistä).
